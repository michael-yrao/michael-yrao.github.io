Same maze, same start and goal — BFS explores level by level and is guaranteed to find the shortest path; DFS commits to one corridor at a time and returns whatever path it stumbles into first. Run both on a few mazes and watch the gap.

How it works

Phase 1 — carving (randomized DFS)

Start at the top-left cell. Walk to a random unvisited neighbor and knock down the wall between you. Dead end? Backtrack to the last cell with an unvisited neighbor. When the stack empties, every cell is reachable and there is exactly one path between any two cells — a perfect maze. This is the same DFS + backtracking shape as exploring an island in Max Area of Island.

That last row is the recognition cue: in an unweighted grid or graph, the words "shortest" or "fewest steps" always mean BFS. DFS is the tool for exhaustive exploration and existence checks.

Why BFS is always shortest

BFS expands in rings by distance. Ring 1 is every cell one step from the start. Ring 2 is every cell two steps away that wasn't already in ring 1. And so on. Think of dropping a stone in still water — ripples spread one layer at a time.

The first time BFS reaches the goal, that reach happens inside the earliest possible ring. If a shorter path existed, the goal would have appeared in a prior ring — but it didn't. So the first touch is guaranteed to be the minimum number of steps.

Each ring is one step further from S. G is first touched at ring 3 — that's the shortest path.

Why DFS can miss shorter paths

DFS picks one direction and follows it all the way to the end before backtracking. If the first corridor it chooses happens to be a long detour, DFS commits to it fully — even if a 3-step shortcut was sitting one cell to the right. It finds a path, not necessarily the shortest one.
