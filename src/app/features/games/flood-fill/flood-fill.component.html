How the Algorithm Works

This game is a direct playable implementation of BFS (Breadth-First Search). The wave ripple you see on each move is the BFS frontier expanding one level at a time.

Game mechanic BFS concept
Conquered region Visited set
Color you pick Target value to explore
Wave ripple animation BFS level (one dequeue round)
Board edges Out-of-bounds guard
Full board conquered All nodes visited

What happens each move

  1. All conquered cells immediately repaint to your chosen color
  2. The queue is seeded with unconquered neighbors that already match that color
  3. Each dequeue round adds new matching neighbors — one wave in the animation
  4. Process repeats until the queue is empty (no more cells can be absorbed)
  5. Win condition: conquered set size equals the total cell count

How the move limit is set

Before you play, a greedy solver runs on the freshly generated board. Each simulated turn it picks the color that absorbs the most cells — a fast, strong heuristic (though not provably optimal, since optimal Flood It is NP-hard). The solver's score becomes par. Your move limit is par + {{ moveBuffer }}, giving you a small buffer to play imperfectly and still win.

greedy solve + {{ moveBuffer }} buffer = {{ maxMoves }} moves
